Chapter Z, Question Z.51.12

Z.51.12.Q—A government is developing an enterprise resource planning (ERP) system with multiple modules, including procurement, human resources, and financial reporting. Each of these modules will be developed on an individual timetable and, as a result, will become operational at different points. Should the guidance in Statement 51 for reporting outlays associated with the development of internally generated computer software be applied to each individual module of the ERP system or, instead, to the ERP system as a whole?
A—The guidance for reporting outlays associated with the development of internally generated computer software in Statement 51 is based on the stages of software development. Generally, in the above circumstance, each module will have its own development cycle, particularly as it relates to application development-stage and post-implementation/operation-stage activities described in paragraph 10 of Statement 51. Therefore, the guidance for internally generated computer software in Statement 51 should be applied for each individual module of the system rather than the system as a whole.
