Chapter Z, Question Z.51.18

Z.51.18.Q—How should outlays associated with the internal modification of an existing computer software system that makes it able to interface with a new computer software system be reported?
A—Such outlays should be accounted for in accordance with paragraphs 11 and 12 of Statement 51 if the modification results in an increase in the functionality or efficiency of the existing computer software or an extension of the estimated useful life of that software. If the modification does not result in any of these outcomes, the modification should be considered maintenance, and the associated outlays should be expensed as incurred.
Conversely, if the new computer system cannot accomplish its purpose without building the interface with the existing computer software, this cost would be deemed necessary to prepare the new computer software for use and would, therefore, be subject to capitalization as part of the new software application development stage. (See Question Z.51.22 for further clarification.)
